    What Went Well âœ…¶

    1. Early PoC Success: GL integration PoC in Month 1 validated technical approach
    2. Requirements Workshop: Identified 15 gaps early, preventing rework
    3. Phased Approach: 5 phases allowed for incremental delivery and feedback
    4. Risk Team Collaboration: Early involvement ensured IFRS 9 accuracy
    5. QA Team Addition: Adding 3 QA members in Month 4 improved quality
    6. User Training: Comprehensive training led to 95% adoption rate

    Areas for Improvement ðŸ”„¶

    1. Documentation: More inline code comments needed (addressed in Month 11)
    2. Bulk Import: Feature requested in Month 4, delivered in Month 7
    3. Mobile App: Should have started earlier (planned for Month 10, high demand)
    4. Performance Testing: Should have been more frequent (added monthly from Month 7)

    Recommendations for Future Projects¶

    1. Start mobile development earlier if user base is mobile-heavy
    2. Conduct performance testing monthly, not just at phase end
    3. Involve operations team earlier for smoother handover
    4. Allocate more time for data migration (took longer than expected)
