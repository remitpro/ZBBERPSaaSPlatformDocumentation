    Team Working Agreements¶

    Core Hours¶

    • Monday - Friday: 9:00 AM - 5:00 PM (local time)
    • Flexible hours: Team members may adjust with lead approval
    • Core collaboration hours: 10:00 AM - 3:00 PM (all must be available)

    Meetings¶

    • Daily Standup: 9:30 AM (15 minutes) - All team members
    • Weekly Team Meeting: Mondays 2:00 PM (60 minutes)
    • Sprint Planning: Every 2 weeks, Mondays 10:00 AM (2 hours)
    • Sprint Retrospective: Every 2 weeks, Fridays 3:00 PM (1 hour)

    Communication Channels¶

    • Urgent Issues: Phone call or instant message
    • General Questions: Team chat channel
    • Formal Communication: Email
    • Documentation: Project folder (SharePoint)
    • Code: Git repository

    Response Time Expectations¶

    • Urgent (P1): 1 hour
    • High Priority (P2): 4 hours
    • Normal (P3): 1 business day
    • Low (P4): 2 business days

    Code Standards¶

    • Code Reviews: Required for all commits (2 approvers)
    • Testing: Unit tests required (>80% coverage)
    • Documentation: Inline comments and README files
    • Branching: Feature branches, no direct commits to main
    • Commit Messages: Descriptive, reference ticket numbers

    Decision Making¶

    • Technical Decisions: Technical Lead (with team input)
    • Functional Decisions: Functional Lead (with stakeholder input)
    • Project Decisions: Project Manager
    • Escalations: PM → Sponsor → Steering Committee

    Conflict Resolution¶

    1. Direct discussion between parties
    2. Escalate to team lead if unresolved
    3. Escalate to PM if still unresolved
    4. PM final decision or escalate to sponsor
