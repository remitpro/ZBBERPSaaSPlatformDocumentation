    Requirements Validation Workshop¶

    ZBB ERP Module Implementation¶

    Workshop Date: [To be scheduled - Month 1, Week 2]
    Duration: Full day (8 hours)
    Location: [Conference Room]
    Facilitator: Functional Lead

    Workshop Objectives¶

    1. Validate functional requirements with key stakeholders
    2. Identify any gaps or missing requirements
    3. Prioritize requirements (Must-have, Should-have, Nice-to-have)
    4. Confirm understanding of business processes
    5. Build consensus on solution approach

    Participants¶

    Required Attendees¶

    • [ ] Head of Budget (Workshop Sponsor)
    • [ ] Budget Team Representatives (3-4 people)
    • [ ] Heads of Business Units (sample of 3-4 units)
    • [ ] Operations Team Lead
    • [ ] Finance Team Representative
    • [ ] Functional Lead (Facilitator)
    • [ ] Business Analysts (3)
    • [ ] Technical Lead

    Optional Attendees¶

    • [ ] Project Manager
    • [ ] Change Management Lead

    Agenda¶

    Session 1: Introduction and Context (9:00 AM - 9:30 AM)¶

    Objectives: - Set workshop context and objectives - Review current state challenges - Introduce FRD structure

    Activities: - Welcome and introductions - Workshop objectives and ground rules - Current process pain points discussion - FRD overview presentation

    Materials: Project Charter, FRD Executive Summary

    Session 2: System Integration & Architecture (9:30 AM - 10:30 AM)¶

    Objectives: - Validate Chart of Accounts mapping approach - Confirm GL integration requirements - Review RBAC model

    Discussion Topics: 1. Chart of Accounts Mapping - Current CoA structure - Multi-dimensional requirements (Cost Center, Business Unit, Product, Geography) - Validation rules

    1. GL Integration
    2. Data synchronization frequency
    3. Integration points

    4. Error handling

    5. User Roles and Permissions

    6. Review 6 user roles (Executive, Budget Team, Unit Heads, Operations, Finance, Auditors)
    7. Validate permissions matrix
    8. Delegation requirements

    Validation Questions: - Is the CoA structure complete and accurate? - Are all necessary dimensions captured? - Are user roles correctly defined? - Any missing permissions or access requirements?

    Materials: FRD Section 2

    Break (10:30 AM - 10:45 AM)¶

    Session 3: Assumptions Engine & ZBB Logic (10:45 AM - 12:00 PM)¶

    Objectives: - Validate assumptions management requirements - Confirm ZBB template design - Review justification requirements

    Discussion Topics:

    1. Assumptions Engine (FR-ASM-001 to FR-ASM-007)
    2. Assumption categories (Macroeconomic, Growth, Regulatory, Operational)
    3. Version control requirements
    4. Approval workflow
    5. Scenario support (Base, Optimistic, Pessimistic)

    Validation Questions: - Are all assumption categories covered? - Is version control approach acceptable? - Who should approve assumption changes?

    1. ZBB Templates (FR-ZBB-001 to FR-ZBB-010)
    2. Template structure and fields
    3. Mandatory justification (50 character minimum)
    4. Document attachment thresholds
    5. Decision Package concept

    Validation Questions: - Is 50 characters sufficient for justifications? - What threshold should trigger mandatory document attachment? ($50K suggested) - Is Decision Package grouping useful? - What business drivers should be in the dropdown?

    Materials: FRD Section 3.1, 3.2

    Lunch Break (12:00 PM - 1:00 PM)¶

    Session 4: Loan Portfolio & Funding Modules (1:00 PM - 2:30 PM)¶

    Objectives: - Validate loan portfolio budget structure - Confirm funding sources and calculations - Review interest income/expense logic

    Discussion Topics:

    1. Loan Portfolio Module (FR-LOAN-001 to FR-LOAN-012)
    2. Product types taxonomy
    3. PFI types and PFI master data
    4. Disbursement template (Product PFI Type PFI Name)
    5. Monthly phasing approach
    6. Interest income calculations
    7. Provisioning logic (IFRS 9)

    Validation Questions: - Are all product types captured? - Is the PFI classification correct? - Is monthly phasing granular enough? - Are interest income formulas accurate? - Is provisioning calculation aligned with accounting standards?

    1. Borrowings & Funding Module (FR-FUND-001 to FR-FUND-007)
    2. Funding source types
    3. Borrowings schedule template
    4. Interest expense calculations
    5. Weighted average cost of funds

    Validation Questions: - Are all funding sources covered? - Is deposit projection approach acceptable? - Are interest expense formulas correct?

    Materials: FRD Section 3.3, 3.4

    Break (2:30 PM - 2:45 PM)¶

    Session 5: Financial Statements & KPIs (2:45 PM - 4:00 PM)¶

    Objectives: - Validate Three Statement Model integration - Confirm KPI formulas - Review reporting requirements

    Discussion Topics:

    1. Three Statement Model (FR-IS, FR-BS, FR-CF)
    2. Income Statement structure
    3. Balance Sheet structure
    4. Cash Flow Statement (indirect method)
    5. Integration validation checks

    Validation Questions: - Is the Income Statement structure complete? - Are all Balance Sheet line items captured? - Is the indirect method acceptable for Cash Flow? - Are integration checks sufficient?

    1. KPI Engine (FR-KPI-001 to FR-KPI-013)
    2. Review all 13 KPI formulas:
      • Net Interest Margin (NIM)
      • Return on Assets (ROA)
      • Return on Equity (ROE)
      • Cost-to-Income Ratio
      • Loan Growth Rate
      • NPL Ratio
      • Provision Coverage Ratio
      • Loan-to-Deposit Ratio (LDR)
      • Liquidity Ratio
      • Capital Adequacy Ratio (CAR)
      • EPS, Dividend Payout, Net Interest Spread

    Validation Questions: - Are formulas mathematically correct? - Are these the right KPIs for management? - Any missing KPIs? - What are acceptable target ranges for each KPI?

    Materials: FRD Sections 4, 5

    Session 6: Workflow, Validation & Reporting (4:00 PM - 5:00 PM)¶

    Objectives: - Validate workflow stages and approvals - Confirm validation rules - Review reporting requirements

    Discussion Topics:

    1. Workflow (FR-WF-001 to FR-WF-010)
    2. 8-stage workflow: Draft Submit Review Query Approve Consolidate Final Review Publish
    3. Approval hierarchy
    4. SLA timers
    5. Notification triggers

    Validation Questions: - Is the workflow appropriate? - Who should be in each approval stage? - What are acceptable SLAs (e.g., 3 days for review)?

    1. Validation Framework (FR-VAL-001 to FR-VAL-019)
    2. Hard validations (9 blocking rules)
    3. Soft validations (6 warnings)
    4. Error messages

    Validation Questions: - Are validation rules appropriate? - Any missing validations? - Are warning thresholds correct (e.g., 20% budget growth)?

    1. Reporting (FR-REP-001 to FR-REP-009)
    2. Standard reports
    3. Variance analysis
    4. Scenario analysis
    5. Export formats

    Validation Questions: - Are all necessary reports covered? - Any missing reports? - Are export formats sufficient (Excel, PDF, CSV)?

    Materials: FRD Sections 6, 7, 8

    Session 7: Wrap-up and Next Steps (5:00 PM - 5:30 PM)¶

    Activities: - Summary of key findings and decisions - Review identified gaps or changes - Prioritization exercise (MoSCoW method) - Must have: Critical for go-live - Should have: Important but not critical - Could have: Nice to have if time/budget permits - Won't have: Out of scope for this phase - Action items and owners - Next steps

    Outputs: - Validated requirements list - Identified gaps/changes - Prioritized requirements - Action item tracker

    • [ ] Request participants to review assigned sections
    • [ ] Prepare workshop presentation materials
    • [ ] Book conference room
    • [ ] Arrange refreshments

    3 Days Before¶

    • [ ] Send reminder with agenda and pre-read materials
    • [ ] Prepare flip charts, sticky notes, markers
    • [ ] Set up projector/screen
    • [ ] Prepare sign-in sheet

    1 Day Before¶

    • [ ] Confirm attendance
    • [ ] Print handouts (agenda, FRD summary, worksheets)
    • [ ] Test equipment

    • [ ] Distribute workshop notes and decisions
    • [ ] Share action item tracker

    • [ ] Document all requirement changes
    • [ ] Update requirements traceability matrix

    Success Criteria¶

    • All key stakeholders participate actively
    • All FRD sections reviewed and validated
    • Gaps and changes documented
    • Requirements prioritized
    • Consensus achieved on solution approach
    • Action items assigned with due dates
    • Updated FRD ready for sign-off within 1 week
