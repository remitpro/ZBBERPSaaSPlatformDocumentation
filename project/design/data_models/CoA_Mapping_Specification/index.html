    Chart of Accounts (CoA) Mapping Specification¶

    ZBB ERP Module Implementation¶

    Document Owner: Functional Lead
    Period: Month 2, Week 1-2
    Status: In Progress

    1. Overview¶

    This document specifies the mapping of the Bank's General Ledger Chart of Accounts to the Budget Module structure, including multi-dimensional mapping requirements identified during the requirements validation workshop.

    4. CoA Hierarchy Structure¶

    5-Level Hierarchy¶

    Level 1: Account Class (1 digit)
   Level 2: Account Category (2 digits)
       Level 3: Account Group (3 digits)
           Level 4: Account Sub-Group (4 digits)
               Level 5: Detail Account (7 digits)

    Example Hierarchy¶

    1 - Assets
   10 - Cash and Bank Balances
      100 - Cash on Hand
         1000 - Cash - Head Office
            1000-001 - Cash - Head Office - Vault
            1000-002 - Cash - Head Office - Teller
         1001 - Cash - Branches
             1001-001 - Cash - Branch A
      101 - Bank Balances
          1010 - Central Bank
              1010-001 - Current Account - Central Bank
   105 - Loans and Advances
       1050 - Loans to Customers
           1050-001 - Retail Loans
           1050-002 - SME Loans
           1050-003 - Corporate Loans

    6. Budget Allocation Model¶

    Allocation Combinations¶

    Budget lines will be allocated using combinations of dimensions:

    Primary Model: 

    Budget Line = GL Account  Cost Center  Business Unit  Product  Geography

    Example Budget Line: - GL Account: 5020-015 (Personnel Expenses - Salaries) - Cost Center: CC-010 (Retail Banking) - Business Unit: BU-010 (Retail Banking Division) - Product: N/A (Overhead, not product-specific) - Geography: GEO-001 (North Region) - Budget Amount: $500,000

    Allocation Rules¶

    1. Revenue Accounts: Must have Product and Geography
    2. Direct Expenses: Must have Product and Geography
    3. Overhead Expenses: Cost Center and Business Unit only
    4. Capital Expenditure: Business Unit only

    7. ERP System Mapping¶

    Target ERP Structure¶

    Account Master Table: 

    CREATE TABLE BudgetAccountMaster (
    AccountID INT PRIMARY KEY,
    GLAccountCode VARCHAR(10) NOT NULL,
    AccountDescription VARCHAR(255),
    AccountType VARCHAR(20), -- Asset, Liability, Equity, Income, Expense
    ParentAccountID INT,
    Level INT, -- 1 to 5
    IsActive BIT,
    IsLeafNode BIT -- True if detail account (can have budget)
);

    Dimension Tables: 

    CREATE TABLE CostCenterMaster (
    CostCenterID INT PRIMARY KEY,
    CostCenterCode VARCHAR(10),
    CostCenterName VARCHAR(100),
    IsActive BIT
);

CREATE TABLE BusinessUnitMaster (
    BusinessUnitID INT PRIMARY KEY,
    BusinessUnitCode VARCHAR(10),
    BusinessUnitName VARCHAR(100),
    IsActive BIT
);

CREATE TABLE ProductMaster (
    ProductID INT PRIMARY KEY,
    ProductCode VARCHAR(10),
    ProductName VARCHAR(100),
    ProductCategory VARCHAR(50),
    IsActive BIT
);

CREATE TABLE GeographyMaster (
    GeographyID INT PRIMARY KEY,
    GeographyCode VARCHAR(10),
    GeographyName VARCHAR(100),
    IsActive BIT
);

    Budget Line Table: 

    CREATE TABLE BudgetLine (
    BudgetLineID INT PRIMARY KEY,
    BudgetHeaderID INT,
    AccountID INT,
    CostCenterID INT,
    BusinessUnitID INT,
    ProductID INT NULL, -- Nullable for overhead
    GeographyID INT NULL, -- Nullable for overhead
    BudgetAmount DECIMAL(18,2),
    Justification VARCHAR(500),
    -- Monthly phasing columns
    Jan DECIMAL(18,2), Feb DECIMAL(18,2), ..., Dec DECIMAL(18,2)
);

    9. Validation Rules¶

    CoA Validation¶

    1. Unique Account Codes: No duplicate GL account codes
    2. Valid Hierarchy: All accounts have valid parent (except Level 1)
    3. Leaf Node Budget: Only leaf nodes can have budget allocations
    4. Active Status: Only active accounts available for budgeting

    Dimension Validation¶

    1. Mandatory Dimensions: GL Account, Cost Center, Business Unit always required
    2. Product Dimension: Mandatory for Income and Direct Expense accounts
    3. Geography Dimension: Mandatory for Income and Direct Expense accounts
    4. Valid Combinations: All dimension values must exist in master tables
